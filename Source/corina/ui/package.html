This isn't used yet (it's just a rough hack), but the idea is to be able to generate parts of the UI automatically, like menubars and menus, and (nested) component layouts. It's sort of like XUL, except that it's converted to actual code and compiled at build-time, so it gives the programmer increased readability without sacrificing any performance at runtime.

For testing, my sample input file ("GridMenus.xmlui") is:

<?xml version="1.0"?>

<ui class="GridMenus" package="corina.cross">

        <menu id="view" title="View" mnemonic="V">
                <menuitem id="graph" title="Graph All"/>
                <separator/>
                <menuitem id="zoom_in" title="Zoom In" mnemonic="I" accelerator="accel up"/>
                <menuitem id="zoom_out" title="Zoom Out" mnemonic="O" accelerator="accel down"/>
        </menu>
</ui>

Which generates this output file ("GridMenus.java"):

package corina.cross;

public class GridMenus {
  public static javax.swing.JComponent getComponent() {
   javax.swing.JMenu id0 = new javax.swing.JMenu("View");
   id0.setMnemonic('V');
   javax.swing.JMenuItem id1 = new javax.swing.JMenuItem("Graph All");
   javax.swing.JMenuItem id3 = new javax.swing.JMenuItem("Zoom In");
   id3.setMnemonic('I');
   id3.setAccelerator(corina.ui.RuntimeUtils.getKeyStroke("accel up"));
   javax.swing.JMenuItem id4 = new javax.swing.JMenuItem("Zoom Out");
   id4.setMnemonic('O');
   id4.setAccelerator(corina.ui.RuntimeUtils.getKeyStroke("accel down"));
   id0.add(id1);
   id0.addSeparator();
   id0.add(id3);
   id0.add(id4);

    return id0;
  }
}
