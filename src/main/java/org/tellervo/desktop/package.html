Event model

The event model consists of 3 classes: SampleEvent, SampleListener, and SampleAdapter. Currently, it only deals with Samples, as those are the basic data type of Corina, but if necessary it can easily be extended to other types (for example, Grids).

Using the event model is simple:

  1. If you're displaying a view of a Sample, implement SampleListener so you'll know when it changes.
  2. If you change a Sample, call one of its fire methods so all the listeners will know to update their views.

That's it.

One last detail: SampleListener is the interface to implement. If you're not extending another class, though, you can extend SampleAdapter, which is just a listener with all empty methods.

Currently, the events are:

  • sampleRedated, which means the starting/ending dates have changed
  • sampleDataChanged, which means that one or more of the numbers of the dataset have changed
  • sampleMetadataChanged, which means that one or more of the metadata fields has been changed (the element changed, or a key was added or removed)
